# 实验问题与解决方案总结

## 一、环境配置问题

### 1. Kafka 权限问题
**问题描述**：
```
java.nio.file.AccessDeniedException: /tmp/kafka-logs/__consumer_offsets-4/00000000000000000000.log
```

**原因**：Kafka 日志目录 `/tmp/kafka-logs/` 属于 root 用户，当前用户 jules 无权访问。

**解决方案**：
```bash
echo '54088Cnm,' | sudo -S chown -R jules:jules /tmp/kafka-logs
echo '54088Cnm,' | sudo -S chown -R jules:jules /tmp/zookeeper
echo '54088Cnm,' | sudo -S chown -R jules:jules /opt/kafka/logs
```

---

### 2. Flume 命令路径问题
**问题描述**：
```
flume-ng: command not found
```

**原因**：`flume-ng` 不在系统 PATH 环境变量中。

**解决方案**：使用完整路径执行
```bash
cd /opt/flume
./bin/flume-ng agent --conf conf --conf-file <config-file> --name a1
```

---

### 3. Flume 日志配置文件权限问题
**问题描述**：
```
ERROR Unable to create file ./flume.log java.io.IOException: Permission denied
```

**原因**：Flume 默认尝试写入日志文件到当前目录，但无权限。

**解决方案**：修改 `/opt/flume/conf/log4j2.xml`，只保留 Console 输出：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Configuration status="ERROR">
  <Appenders>
    <Console name="Console" target="SYSTEM_OUT">
      <PatternLayout pattern="%d (%t) [%p - %l] %m%n" />
    </Console>
  </Appenders>
  <Loggers>
    <Logger name="org.apache.flume.lifecycle" level="info"/>
    <Logger name="org.jboss" level="WARN"/>
    <Logger name="org.apache.avro.ipc.netty.NettyTransceiver" level="WARN"/>
    <Logger name="org.apache.hadoop" level="INFO"/>
    <Logger name="org.apache.hadoop.hive" level="ERROR"/>
    <Root level="INFO">
      <AppenderRef ref="Console" />
    </Root>
  </Loggers>
</Configuration>
```

## 二、数据源配置问题

### 4. Flume JDBC Source 不可用
**问题描述**：
```
java.lang.ClassNotFoundException: org.apache.flume.source.jdbc.JDBCSource
```

**原因**：Flume 默认不包含 JDBC Source，需要额外安装插件。

**解决方案**：使用 Exec Source + Python 脚本替代
```bash
# 创建读取 MySQL 的 Python 脚本
python3 mysql_to_stdout.py  # 输出 JSON 到 stdout

# Flume 配置
a1.sources.exec-source.type = exec
a1.sources.exec-source.command = /tmp/workspace/mysql_to_kafka_loop.py
```


**原因**：Flume 的 Java 进程无法正确处理中文路径（`/mnt/d/s/作业`）。

### 6. Flume Exec Source 只执行一次就退出
**问题描述**：
```
Command [/tmp/workspace/run_mysql_to_kafka.sh] exited with 0
```

**原因**：
1. 脚本执行完成后立即退出
2. 虚拟环境激活在 Flume 子进程中不生效

**解决方案**：创建持续运行的独立 Python 脚本
```python
#!/usr/bin/env python3
import json
import pymysql
import time

def get_students():
    connection = pymysql.connect(
        host='localhost',
        port=3307,
        user='testuser',
        password='testpass',
        database='school',
        charset='utf8mb4'
    )
    try:
        with connection.cursor() as cursor:
            cursor.execute('SELECT id, name, age, grade FROM student')
            return cursor.fetchall()
    finally:
        connection.close()

if __name__ == '__main__':
    while True:
        results = get_students()
        for row in results:
            record = {
                'id': row[0],
                'name': row[1],
                'age': row[2],
                'grade': row[3]
            }
            print(json.dumps(record))
        time.sleep(10)  # 每10秒执行一次
```

## 三、HDFS 集成问题

### 7. Flume HDFS Sink 缺少依赖
**问题描述**：
```
java.lang.NoClassDefFoundError: org/apache/hadoop/io/SequenceFile$CompressionType
```

**原因**：Flume 启动时未加载 Hadoop 依赖库。

**解决方案**：创建 `/opt/flume/conf/flume-env.sh`
```bash
export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin
```

如果仍有问题，使用 Python 替代方案：
```python
from kafka import KafkaConsumer
import subprocess

consumer = KafkaConsumer('flume-input-topic', ...)
for message in consumer:
    subprocess.run(['/usr/local/hadoop/bin/hdfs', 'dfs', '-put', '-f',
                   local_file, hdfs_path])
```

## 四、Kafka 消费问题

### 8. Kafka 消费者无数据接收
**问题描述**：
```
Kafka消费者已启动，等待Flume传递的数据...
控制台输出完成！
```

**原因**：
1. 消费者设置了 `consumer_timeout_ms`，超时后退出
2. 新消费者使用了 group_id，可能跳过已消费消息

**解决方案**：
1. 使用 `auto_offset_reset='earliest'`
2. 使用 Kafka 自带消费者验证数据存在
```bash
cd /opt/kafka
bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 \
    --topic flume-mysql-topic --from-beginning
```
