## 六、常用命令参考

### Kafka 命令
```bash
# 查看 Topic 列表
cd /opt/kafka
bin/kafka-topics.sh --list --bootstrap-server localhost:9092

# 创建 Topic
bin/kafka-topics.sh --create --topic <topic-name> --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1

# 删除 Topic
bin/kafka-topics.sh --delete --topic <topic-name> --bootstrap-server localhost:9092

# 控制台生产者
bin/kafka-console-producer.sh --bootstrap-server localhost:9092 --topic <topic-name>

# 控制台消费者
bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic <topic-name> --from-beginning
```

### Flume 命令
```bash
# 启动 Agent
cd /opt/flume
flume-ng agent --conf conf --conf-file <conf-file-path> --name <agent-name> -Dflume.root.logger=INFO,console

# 查看帮助
flume-ng help
```

### MySQL 命令
```bash
# 连接MySQL
mysql -h localhost -P 3307 -u testuser -ptestpass

# 连接MySQL（root用户）
mysql -h localhost -P 3307 -u root -p

# 查看数据库列表
show databases;

# 使用数据库
use testdb;

# 查看表列表
show tables;

# 查询数据
select * from student;
```

## 七、故障排除

### 7.1 Kafka 无法启动
- 检查端口 2181（Zookeeper）和 9092（Kafka）是否被占用
- 检查目录权限：
  ```bash
  echo '54088Cnm,' | sudo -S chown -R jules:jules /tmp/kafka-logs
  echo '54088Cnm,' | sudo -S chown -R jules:jules /tmp/zookeeper
  echo '54088Cnm,' | sudo -S chown -R jules:jules /opt/kafka/logs
  ```
- 使用 `restart_kafka_full.sh` 脚本重启服务

### 7.2 MySQL 连接失败
- 检查 MySQL 服务状态：`systemctl status mysql`
- 验证端口是否正确（本实验使用 3307）
- 确认用户名密码是否正确

### 7.3 Flume 连接失败
- 确保 Kafka 服务正常运行
- 检查配置文件中的 `kafka.bootstrap.servers` 配置
- 确保 Flume 配置文件路径正确

### 7.4 Python 模块找不到
- 确保已激活虚拟环境：`source venv/bin/activate`
- 重新安装依赖：`pip install kafka-python pymysql pymongo pandas`
