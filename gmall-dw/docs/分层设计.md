
# 数仓分层设计

## 一、分层架构概述

| 层级 | 名称 | 职责 | 存储格式 | 表命名规范 |
|------|------|------|---------|-----------|
| ODS | 原始数据层 | 直接同步业务库与日志，不清洗、不加工 | Text | ods_* |
| DIM | 公共维度层 | 用户、商品、店铺、时间等维度表 | ORC | dim_* |
| DWD | 明细数据层 | 清洗、脱敏、去重、统一规范 | ORC | dwd_* |
| DWS | 汇总数据层 | 按主题轻度聚合（用户、订单、商品等） | ORC | dws_* |
| ADS | 应用数据层 | 指标报表、大屏、接口输出 | ORC | ads_* |

## 二、各层详细设计

### 2.1 ODS层（原始数据层）

**职责**：
- 直接从业务数据库同步数据
- 直接采集日志数据
- 保持原始数据格式，不做任何加工
- 用于数据追溯和审计

**表设计原则**：
- 与源表结构保持一致
- 使用Text格式存储
- 按天分区（dt）

**典型表**：
- ods_user_info - 用户信息表
- ods_order_info - 订单信息表
- ods_order_detail - 订单明细表
- ods_sku_info - 商品SKU表
- ods_log - 日志表

### 2.2 DIM层（公共维度层）

**职责**：
- 存储维度数据
- 处理缓慢变化维（SCD）
- 提供统一的维度编码

**表设计原则**：
- 使用ORC压缩格式
- 拉链表设计（SCD Type 2）
- 存储历史快照

**典型表**：
- dim_user - 用户维度表（拉链表）
- dim_sku - 商品维度表
- dim_time - 日期维度表
- dim_province - 地区维度表

### 2.3 DWD层（明细数据层）

**职责**：
- 数据清洗（去重、过滤脏数据）
- 数据脱敏（手机号、邮箱等）
- 格式统一（时间格式、编码统一）
- 关联维度键

**表设计原则**：
- 使用ORC压缩格式
- 事务型事实表
- 保留明细数据

**典型表**：
- dwd_order_detail - 订单明细事实表
- dwd_payment - 支付事实表
- dwd_log_page - 页面日志事实表

### 2.4 DWS层（汇总数据层）

**职责**：
- 按主题轻度聚合
- 按维度汇总指标
- 提供跨主题关联

**表设计原则**：
- 使用ORC压缩格式
- 按天/周/月聚合
- 多粒度汇总

**典型表**：
- dws_user_day - 用户每日汇总
- dws_order_day - 订单每日汇总
- dws_sku_day - 商品每日汇总

### 2.5 ADS层（应用数据层）

**职责**：
- 计算最终指标
- 为报表和大屏提供数据
- 提供API接口数据

**表设计原则**：
- 使用ORC压缩格式
- 面向业务指标
- 多时间粒度（1日/7日/30日）

**典型表**：
- ads_gmv_day - GMV日报表
- ads_user_retention - 用户留存表
- ads_conversion_rate - 转化率表

## 三、分区策略

| 层级 | 分区方式 | 说明 |
|------|---------|------|
| ODS | dt (日期) | 按天分区 |
| DIM | dt (日期) | 拉链表按天更新 |
| DWD | dt (日期) | 按天分区 |
| DWS | dt (日期) | 按天分区 |
| ADS | 无分区 | 全量表 |

## 四、数据生命周期

| 层级 | 保留周期 | 说明 |
|------|---------|------|
| ODS | 3个月 | 可根据业务需求调整 |
| DIM | 永久 | 维度数据需长期保存 |
| DWD | 1个月 | 明细数据可定期清理 |
| DWS | 永久 | 汇总数据需长期保存 |
| ADS | 永久 | 指标数据需长期保存 |
