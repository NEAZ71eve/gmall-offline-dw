# 电商离线数据仓库运维手册

## 一、概述

本文档为电商离线数据仓库项目的运维手册，涵盖日常运维、故障排查、性能监控、数据治理等方面。

## 二、日常运维流程

### 2.1 每日运维检查清单

| 序号 | 检查项 | 检查内容 | 责任人 |
|------|-------|---------|--------|
| 1 | 集群状态 | HDFS、YARN、Hive Metastore 状态 | 运维工程师 |
| 2 | ETL任务 | DolphinScheduler工作流执行状态 | 数据工程师 |
| 3 | 数据质量 | 数据完整性、准确性检查 | 数据工程师 |
| 4 | 磁盘空间 | HDFS存储使用率 | 运维工程师 |
| 5 | 日志监控 | 各组件日志异常检查 | 运维工程师 |
| 6 | 告警通知 | 失败告警、超时告警处理 | 值班人员 |

### 2.2 每日运维脚本

```bash
#!/bin/bash

echo "=== 每日运维检查 ==="

# 1. 检查HDFS状态
echo "1. HDFS状态检查"
hdfs dfsadmin -report

# 2. 检查YARN状态
echo "2. YARN状态检查"
yarn node -list

# 3. 检查Hive Metastore
echo "3. Hive Metastore状态"
hive --service metastore --status

# 4. 检查磁盘空间
echo "4. HDFS磁盘使用率"
hdfs dfs -df -h /

# 5. 检查ETL任务状态（调用DolphinScheduler API）
echo "5. ETL任务状态检查"
curl -s -X GET "http://dolphinscheduler:12345/dolphinscheduler/projects/gmall/workflow-instances" \
  -H "token: ${TOKEN}" | jq '.data.total'

# 6. 检查数据质量
echo "6. 数据质量检查"
python /opt/gmall/utils/data_quality_check.py --date $(date -d "yesterday" +%Y-%m-%d)
```

## 三、ETL任务运维

### 3.1 任务调度配置

**工作流名称**：gmall_etl_daily
**调度时间**：每日凌晨2:00
**重试次数**：2次
**重试间隔**：5分钟
**超时时间**：2小时

### 3.2 任务依赖关系

```
DataX同步 → DIM层构建 → DWD层清洗 → DWS层汇总 → ADS层指标 → 数据质量检查 → 通知
```

### 3.3 任务重跑流程

#### 3.3.1 单任务重跑

```bash
# 通过DolphinScheduler API重跑单个任务
curl -X POST "http://dolphinscheduler:12345/dolphinscheduler/projects/gmall/executors/retry" \
  -H "Content-Type: application/json" \
  -H "token: ${TOKEN}" \
  -d '{
    "taskInstanceId": 12345,
    "failureStrategy": "CONTINUE"
  }'
```

#### 3.3.2 批量补数

```bash
# 补数日期范围
START_DATE="2024-01-01"
END_DATE="2024-01-31"

# 遍历日期执行补数
current_date=$START_DATE
while [ "$current_date" != "$END_DATE" ]; do
    echo "补数日期: $current_date"
    
    curl -X POST "http://dolphinscheduler:12345/dolphinscheduler/projects/gmall/executors/start" \
      -H "Content-Type: application/json" \
      -H "token: ${TOKEN}" \
      -d '{
        "scheduleTime": "'"$current_date"' 02:00:00",
        "failureStrategy": "CONTINUE",
        "warningType": "FAILURE"
      }'
    
    current_date=$(date -d "$current_date +1 day" +%Y-%m-%d)
    sleep 60
done
```

## 四、数据质量管理

### 4.1 数据质量检查项

| 检查类型 | 检查内容 | 检查方法 | 告警级别 |
|---------|---------|---------|---------|
| 完整性 | 数据行数是否为0 | 统计COUNT | ERROR |
| 准确性 | 数值字段是否在合理范围 | 统计MIN/MAX | WARN |
| 一致性 | 关联数据是否匹配 | JOIN验证 | ERROR |
| 唯一性 | 是否存在重复数据 | COUNT(DISTINCT) | WARN |
| 时效性 | 数据是否按时到达 | 时间戳检查 | ERROR |

### 4.2 数据质量检查脚本

```python
#!/usr/bin/env python3
# data_quality_check.py

import sys
from pyhive import hive

def check_data_quality(biz_date):
    conn = hive.connect(host='localhost', port=10000)
    cursor = conn.cursor()
    
    checks = [
        {
            'name': 'ODS用户数据完整性',
            'sql': f"SELECT COUNT(*) FROM gmall_ods.ods_user_info WHERE dt='{biz_date}'",
            'expected_min': 1,
            'level': 'ERROR'
        },
        {
            'name': 'DWD订单金额合理性',
            'sql': f"SELECT MIN(total_amount), MAX(total_amount) FROM gmall_dwd.dwd_order_info WHERE dt='{biz_date}'",
            'expected_min': 0,
            'expected_max': 1000000,
            'level': 'WARN'
        },
        {
            'name': 'DWS汇总数据一致性',
            'sql': f"SELECT SUM(order_amount) FROM gmall_dws.dws_order_day WHERE dt='{biz_date}'",
            'compare_sql': f"SELECT SUM(final_amount) FROM gmall_dwd.dwd_order_detail WHERE dt='{biz_date}'",
            'level': 'ERROR'
        }
    ]
    
    all_passed = True
    for check in checks:
        cursor.execute(check['sql'])
        result = cursor.fetchone()
        
        if check.get('expected_min') and result[0] < check['expected_min']:
            print(f"❌ [{check['level']}] {check['name']}: 最小值校验失败")
            all_passed = False
        elif check.get('expected_max') and result[0] > check['expected_max']:
            print(f"❌ [{check['level']}] {check['name']}: 最大值校验失败")
            all_passed = False
        else:
            print(f"✅ [{check['level']}] {check['name']}: 检查通过")
    
    cursor.close()
    conn.close()
    return all_passed

if __name__ == '__main__':
    if len(sys.argv) < 2:
        print("Usage: python data_quality_check.py <biz_date>")
        sys.exit(1)
    
    biz_date = sys.argv[1]
    if check_data_quality(biz_date):
        sys.exit(0)
    else:
        sys.exit(1)
```

## 五、故障排查

### 5.1 常见故障及排查方法

| 故障类型 | 现象 | 排查方法 | 解决措施 |
|---------|------|---------|---------|
| HDFS存储空间不足 | Hive任务写入失败 | `hdfs dfs -df -h` | 清理过期数据或扩容 |
| YARN资源不足 | 任务无法分配容器 | `yarn top` | 调整资源配置或等待 |
| Hive Metastore连接失败 | Hive命令执行失败 | 检查Metastore服务状态 | 重启Metastore |
| DataX同步失败 | ODS层数据为空 | 检查DataX日志和数据库连接 | 修复数据源连接 |
| 数据倾斜 | 任务长时间卡在99% | 查看任务日志和执行计划 | 优化SQL或调整数据分布 |
| 网络超时 | 任务执行超时 | 检查网络连通性 | 增加超时时间或优化网络 |

### 5.2 日志查看指南

## 六、性能监控

### 6.1 监控指标

| 指标类型 | 监控项 | 阈值 | 告警方式 |
|---------|-------|------|---------|
| HDFS | 磁盘使用率 | >90% | 钉钉+邮件 |
| YARN | 内存使用率 | >85% | 钉钉 |
| Hive | 查询执行时间 | >30分钟 | 钉钉 |
| DolphinScheduler | 任务失败率 | >5% | 钉钉+邮件 |
| 数据延迟 | ETL完成时间 | >4小时 | 钉钉+邮件 |

### 6.2 监控脚本

```bash
#!/bin/bash

# 检查HDFS使用率
HDFS_USAGE=$(hdfs dfs -df -h / | grep -v "Filesystem" | awk '{print $5}' | sed 's/%//')
if [ $HDFS_USAGE -gt 90 ]; then
    echo "HDFS使用率超过90%: ${HDFS_USAGE}%" | mail -s "HDFS告警" admin@example.com
fi

# 检查YARN内存使用率
YARN_MEM=$(yarn node -list 2>/dev/null | grep -E "GB" | awk '{sum += $5} END {print sum}')
if [ $YARN_MEM -gt 85 ]; then
    echo "YARN内存使用率过高" | mail -s "YARN告警" admin@example.com
fi

# 检查ETL任务状态
ETL_STATUS=$(curl -s -X GET "http://dolphinscheduler:12345/dolphinscheduler/projects/gmall/workflow-instances?startDate=$(date +%Y-%m-%d)" \
  -H "token: ${TOKEN}" | jq '.data.total')
if [ $ETL_STATUS -eq 0 ]; then
    echo "今日ETL任务未执行" | mail -s "ETL告警" admin@example.com
fi
```

## 七、数据生命周期管理

### 7.1 数据保留策略

| 层级 | 保留周期 | 清理策略 | 执行时间 |
|------|---------|---------|---------|
| ODS | 3个月 | 每月1日清理 | 00:00 |
| DWD | 1个月 | 每月1日清理 | 00:30 |
| DWS | 永久 | 不清理 | - |
| ADS | 永久 | 不清理 | - |
| DIM | 永久 | 不清理 | - |

### 7.2 数据清理脚本

```bash
#!/bin/bash

# ODS层清理（保留90天）
echo "清理ODS层90天前数据..."
hive -e "ALTER TABLE gmall_ods.ods_user_info DROP IF EXISTS PARTITION(dt < '$(date -d "90 days ago" +%Y-%m-%d)')"
hive -e "ALTER TABLE gmall_ods.ods_order_info DROP IF EXISTS PARTITION(dt < '$(date -d "90 days ago" +%Y-%m-%d)')"
hive -e "ALTER TABLE gmall_ods.ods_order_detail DROP IF EXISTS PARTITION(dt < '$(date -d "90 days ago" +%Y-%m-%d)')"

# DWD层清理（保留30天）
echo "清理DWD层30天前数据..."
hive -e "ALTER TABLE gmall_dwd.dwd_order_detail DROP IF EXISTS PARTITION(dt < '$(date -d "30 days ago" +%Y-%m-%d)')"
hive -e "ALTER TABLE gmall_dwd.dwd_order_info DROP IF EXISTS PARTITION(dt < '$(date -d "30 days ago" +%Y-%m-%d)')"

echo "数据清理完成"
```

### 7.3 冷热数据迁移

```bash
#!/bin/bash

# 热数据迁移到温数据（7天前）
echo "迁移7天前数据到温存储..."
hive -e "INSERT OVERWRITE TABLE gmall_dwd.dwd_order_detail_warm PARTITION(dt)
         SELECT * FROM gmall_dwd.dwd_order_detail_hot 
         WHERE dt >= '$(date -d "30 days ago" +%Y-%m-%d)' 
           AND dt < '$(date -d "7 days ago" +%Y-%m-%d)'"

# 温数据迁移到冷数据（30天前）
echo "迁移30天前数据到冷存储..."
hive -e "INSERT OVERWRITE TABLE gmall_dwd.dwd_order_detail_cold PARTITION(dt)
         SELECT * FROM gmall_dwd.dwd_order_detail_warm 
         WHERE dt < '$(date -d "30 days ago" +%Y-%m-%d)'"

# 删除冷数据（180天前）
echo "删除180天前冷数据..."
hive -e "ALTER TABLE gmall_dwd.dwd_order_detail_cold DROP IF EXISTS PARTITION(dt < '$(date -d "180 days ago" +%Y-%m-%d)')"
```

## 九、备份与恢复

### 9.1 数据备份策略

| 数据类型 | 备份频率 | 保留周期 | 备份方式 |
|---------|---------|---------|---------|
| DIM层 | 每日 | 7天 | HDFS快照 |
| DWS层 | 每日 | 7天 | HDFS快照 |
| ADS层 | 每日 | 30天 | HDFS快照 + 外部存储 |
| MySQL元数据 | 每日 | 30天 | mysqldump |

### 9.2 备份脚本

```bash
#!/bin/bash

# HDFS快照备份
hdfs dfs -createSnapshot /warehouse/gmall/dim dim_snapshot_$(date +%Y%m%d)
hdfs dfs -createSnapshot /warehouse/gmall/dws dws_snapshot_$(date +%Y%m%d)

# MySQL元数据备份
mysqldump -u root -p hive metastore > /backup/hive_metastore_$(date +%Y%m%d).sql

# 清理7天前快照
hdfs dfs -deleteSnapshot /warehouse/gmall/dim dim_snapshot_$(date -d "7 days ago" +%Y%m%d)
```

### 9.3 恢复流程

```bash
# 从快照恢复
hdfs dfs -renameSnapshot /warehouse/gmall/dim dim_snapshot_20240101 restore_20240101
hdfs dfs -cp /warehouse/gmall/dim/.snapshot/restore_20240101/* /warehouse/gmall/dim/

# MySQL元数据恢复
mysql -u root -p hive < /backup/hive_metastore_20240101.sql
```
