## 摘要

本系统在 Ubuntu 环境下,基于 Docker Compose 容器化编排搭建了一套电商(gmall)大数据平台,覆盖 Hadoop(HDFS/YARN)、Zookeeper、Hive、Spark、MySQL、Superset 六大组件,通过 **Shell 脚本最大化自动化**实现了"环境初始化 → 数据模拟/采集 → 分布式存储 → 离线计算 → 结果入库 → 数据可视化 → 集群自动化运维"的标准大数据工程全流程。系统采用 Hive 5 层数仓模型(ODS→DIM→DWD→DWS→ADS)进行离线计算,Spark 批作业补充深度分析指标,最终通过 Superset 实现可视化看板。集群部署、服务启停、任务调度、故障自愈、日志清理、容器编排等全生命周期运维操作均通过 Shell 脚本自动化实现。

**关键词:** Linux, Docker, Shell自动化, Hadoop, Hive, Spark, 数仓分层, Superset, 离线计算, 运维自动化

## 一、系统概述

### 1.1 选题背景

本课题面向《Linux系统开发》课程考核要求,立足"Linux 底层运维 + 大数据全链路工程落地"核心定位。电商业务每天产生海量订单、用户、商品数据,需要通过大数据平台进行清洗、存储、计算和可视化,为运营决策提供数据支撑。本系统以电商 GMall 为业务域,构建从数据模拟到可视化看板的全链路数据平台。

### 1.2 系统功能结构

```
电商大数据全链路系统
├── 1. 环境初始化 (Shell自动化)
│   ├── 集群服务启动/停止
│   ├── HDFS 目录创建
│   ├── Hive 5 层库初始化
│   └── MySQL 业务库初始化
├── 2. 数据模拟/采集
│   ├── 业务数据模拟 (MySQL init.sql)
│   └── 数据采集入 HDFS (ingest_mysql_to_hdfs.sh)
├── 3. 分布式存储 (HDFS)
│   ├── /origin_data 原始数据存储
│   └── /warehouse 数仓分层存储
├── 4. 离线计算 (Hive + Spark)
│   ├── ODS→DIM 维度加工
│   ├── ODS→DWD 明细加工
│   ├── DWD→DWS 汇总宽表
│   ├── DWS→ADS 应用报表
│   └── Spark 补充指标 (热门商品/转化漏斗)
├── 5. 结果入库 (Hive ADS→MySQL)
│   ├── GMV 日报
│   ├── 热门商品 Top10
│   ├── 用户留存分析
│   └── 转化漏斗
├── 6. 数据可视化 (Superset)
│   ├── GMV 趋势图 (折线)
│   ├── 热门商品排行 (条形)
│   ├── 转化漏斗图
│   ├── 用户留存 (堆叠柱)
│   └── 支付方式分布 (饼图)
└── 7. 集群自动化运维 (Shell)
    ├── 健康巡检
    ├── 故障自愈
    ├── 日志清理
    └── 定时调度
```

### 1.3 系统特点

1. **全容器化部署**: 14 个 Docker 容器统一编排,一键启动/停止
2. **Shell 最大化自动化**: 全部安装、部署、启停、调度、监控、恢复操作通过 Shell 脚本实现
3. **标准数仓分层**: ODS→DIM→DWD→DWS→ADS 五层架构,数据清洗、去重、聚合、汇总流程完整
4. **双计算引擎**: Hive MR 为主计算引擎,Spark 为补充分析引擎,覆盖离线计算场景
5. **Shell 运维体系**: 健康巡检、故障自愈、日志清理、定时调度四维运维能力

## 三、核心架构设计

### 3.1 数据流设计

```
MySQL(business) ──ingest──▶ HDFS(/origin_data) ──ods_load──▶ Hive ODS
                                                                   │
                                              ┌────────────────────┤
                                              ▼                    ▼
                                            DIM                 DWD
                                        (维度表)              (明细事实)
                                              │                    │
                                              └────────┬───────────┘
                                                       ▼
                                                      DWS
                                                 (汇总宽表)
                                                       │
                                              ┌────────┴────────┐
                                              ▼                 ▼
                                           ADS(Hive)      Spark 分析
                                              │                 │
                                              ▼                 ▼
                                    MySQL(ads_*表) ←──────────┘
                                              │
                                              ▼
                                          Superset
                                       (可视化看板)
```

### 3.2 数仓分层设计

| 层 | 库名 | 表数量 | 说明 |
|---|---|---|---|
| ODS | gmall_ods | 4 | 原始数据层,外部表映射 HDFS 源文件 |
| DIM | gmall_dim | 2 | 公共维度层,用户维度/商品维度拉链表 |
| DWD | gmall_dwd | 3 | 明细数据层,订单明细/支付/退款事实 |
| DWS | gmall_dws | 2 | 汇总数据层,日粒度按用户/商品聚合宽表 |
| ADS | gmall_ads | 5+ | 应用数据层,GMV日报/热门商品/留存/漏斗 |

### 3.3 Docker 容器拓扑

```
docker-compose.yml (gmall-net) ─ 14 个容器
  ├── HDFS: namenode, datanode1, datanode2
  ├── YARN: resourcemanager, nodemanager, historyserver
  ├── ZK:   zookeeper
  ├── Hive: hive-metastore-postgresql, hive-metastore, hive-server
  ├── MySQL: mysql
  ├── Spark: spark-master, spark-worker
  └── Superset: superset
```

## 四、Shell 自动化脚本体系 (核心强制要求)

### 4.1 脚本清单

| 编号 | 脚本 | Shell 行数 | 功能 |
|---|---|---|---|
| 1 | `cluster.sh` | 45 | 总控入口,分发到各子脚本 |
| 2 | `start_all.sh` | 70 | 启动全部容器 + 健康检查轮询 |
| 3 | `stop_all.sh` | 10 | 停止全部容器 (保留数据卷) |
| 4 | `cluster_init.sh` | 75 | HDFS目录+Hive库+建表+MySQL校验 |
| 5 | `status.sh` | 100 | 容器/HDFS/Hive/MySQL/Spark/Superset 全巡检 |
| 6 | `fault_recover.sh` | 45 | 宕机检测 + 容器重启自愈 |
| 7 | `log_cleanup.sh` | 85 | 容器日志截断 + HDFS旧分区回收 |
| 8 | `schedule_etl.sh` | 75 | 定时调度 + 一键全链路ETL |
| 9 | `gen_mock_data.sh` | 25 | 数据校验 |
| 10 | `ingest_mysql_to_hdfs.sh` | 50 | MySQL→HDFS 数据采集 |
| 11 | `run_offline_etl.sh` | 90 | Hive 5层ETL执行 |
| 12 | `run_spark_job.sh` | 75 | Spark作业提交 |
| 13 | `export_ads_to_mysql.sh` | 80 | Hive→MySQL 结果导出 |
| 14 | `superset_setup.sh` | 80 | Superset管理员+DB+数据源初始化 |
| **合计** | | **~905行** | |

### 4.2 Shell 编程技术要点

- **函数模块化**: 每个脚本单一职责,`cluster.sh` 总控分发
- **错误处理**: `set -euo pipefail` 严格模式
- **健康检查**: 轮询等待 (`sleep` + 计数器),非固定 `sleep`
- **参数化**: 业务日期 `biz_date` 通过命令行参数传递
- **颜色输出**: ANSI 颜色码区分 INFO/WARN/ERROR
- **日志验证**: 每步执行后 `SELECT COUNT(*)` 校验行数
- **Cron 集成**: `schedule_etl.sh --cron` 输出可注册的 crontab 示例
