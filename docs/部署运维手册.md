# gmall 电商大数据平台 — 部署运维手册

## 一、环境要求

| 项目 | 最低要求 | 推荐 |
|---|---|---|
| 操作系统 | Ubuntu 20.04+ (WSL2 / 原生) | Ubuntu 22.04+ |
| Docker | 20.10+ | 24.0+ |
| Docker Compose | 2.0+ | 2.20+ |
| 内存 | 12 GB | 16 GB+ |
| 磁盘空闲 | 10 GB | 20 GB+ |
| Shell | Bash 4.4+ | Bash 5.x |

## 二、部署

### 2.1 启动 Docker 引擎

**WSL (推荐):**
```bash
# 确保 WSL systemd 已启用 (/etc/wsl.conf 含 boot.systemd=true)
sudo systemctl start docker
docker info  # 验证
```

**Docker Desktop (Windows):**
启动 Docker Desktop → 设置 WSL Integration → 勾选 Ubuntu → Apply & Restart。

### 2.2 一键部署

```bash
cd /path/to/作业

# 1. 拉取镜像 (首次约10-15分钟,视网络)
docker compose pull

# 2. 启动全部服务
./scripts/cluster.sh start

# 3. 等待服务就绪后,初始化环境
./scripts/cluster.sh init

# 4. 巡检确认
./scripts/cluster.sh status
```

### 2.3 验证各组件

```bash
# HDFS
curl http://localhost:9870             # NameNode Web UI
docker exec namenode hdfs dfsadmin -report

# YARN
curl http://localhost:8088             # ResourceManager Web UI

# Hive
docker exec -it hive-server beeline -u jdbc:hive2://localhost:10000
# > SHOW DATABASES;

# MySQL
docker exec -it mysql mysql -u gmall -pgmall123 gmall
mysql> SELECT COUNT(*) FROM user_info;

# Spark
curl http://localhost:18080            # Spark Master Web UI

# Superset
curl http://localhost:8089/health      # 健康检查
# 浏览器打开 http://localhost:8089
```

## 三、执行离线 ETL 全链路

```bash
# 一键执行全链路 (默认 biz_date = 昨天)
./scripts/cluster.sh etl

# 或指定日期
./scripts/schedule_etl.sh --run-now --date 2024-01-22

# 全链路步骤:
# ① cluster_init.sh  → 环境检查和表就绪
# ② ingest_mysql_to_hdfs.sh  → MySQL 业务数据入 HDFS
# ③ run_offline_etl.sh → Hive ODS→DIM→DWD→DWS→ADS
# ④ run_spark_job.sh → Spark 离线补充分析
# ⑤ export_ads_to_mysql.sh → ADS 结果导出 MySQL
```

## 五、运维操作

### 5.1 日常巡检

```bash
./scripts/cluster.sh status
# 输出: 容器状态 + HDFS健康 + Hive连通 + MySQL数据量 + Spark/Superset状态
```

### 5.2 故障自愈

```bash
# 模拟故障: 手动停一个容器
docker stop datanode1

# 执行自愈
./scripts/cluster.sh recover
# 自动检测 non-running 容器并 docker compose up -d 恢复

# 验证
./scripts/cluster.sh status
```

### 5.3 日志查看

```bash
# 关键容器日志
docker logs --tail 50 namenode
docker logs --tail 50 hive-server
docker logs --tail 50 spark-master

# 容器日志摘要
./scripts/cluster.sh logs
```

### 5.4 日志清理

```bash
# 清理容器日志 + HDFS 旧分区 (保留7天)
./scripts/log_cleanup.sh

# Dry-run
./scripts/log_cleanup.sh --dry-run

# 指定保留天数
./scripts/log_cleanup.sh --retain-days 30
```

### 5.5 定时调度

```bash
# 查看 crontab 注册示例
./scripts/schedule_etl.sh --cron

# 注册到 crontab
crontab -e
# 粘贴输出中的行,修改路径

# 验证
crontab -l
```

### 5.6 重启与停止

```bash
# 重启全部
./scripts/cluster.sh restart

# 停止 (保留数据卷)
./scripts/cluster.sh stop

# 彻底清理 (删除所有数据卷!)
docker compose down -v
```

## 六、故障排查

| 现象 | 排查步骤 |
|---|---|
| 容器无法启动 | `docker compose ps` 查看状态; `docker logs <container>` 查看日志 |
| HDFS 无法写入 | `hdfs dfsadmin -report` 看 DataNode 是否 Live; 检查磁盘空间 |
| Hive 查询失败 | `beeline -u jdbc:hive2://localhost:10000 -e "SELECT 1;"` 测试连通; 看 metastore 日志 |
| MySQL 连接拒绝 | `docker exec mysql mysqladmin ping` 看 MySQL 健康; 等 healthcheck 通过 |
| Spark 作业失败 | Spark Master UI (localhost:18080) 查看 Application 日志 |
| Superset 502 | `docker logs superset` 看是否还在初始化; gunicorn 启动约需30-60秒 |
| 内存不足 | 停掉非关键容器: `docker compose stop spark-worker historyserver`; 降低 YARN 内存 |
| 镜像拉取失败 | 切换 docker 镜像源; 或预下载: `docker compose pull` |
